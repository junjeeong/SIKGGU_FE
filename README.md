# [PRD] 식구 (SIKGGU): 식재료를 구출하라!

## 1. 프로젝트 개요

- **서비스명:** 식구 (SIKGGU) 1
- **슬로건:** 식구, 식재료를 구출하라! 2
- **정의:** 유통기한 임박 식재료 폐기 문제와 1인 가구의 고물가 문제를 '위치 기반 실시간 타임세일'로 연결하여 해결하는 O2O 커머스 서비스입니다3.

---

## 2. 타겟 및 페인 포인트 (Pain Point)

- **마트 사장님:** 반품이 불가능한 신선식품이 오늘 판매되지 않으면 그대로 쓰레기가 되는 손실이 발생합니다4.
- **자취생/1인 가구:** 소량의 식재료를 구매하고 싶으나 치솟는 물가에 부담을 느낍니다5.
- **차별점:** 기존 외식/완제품 위주 서비스와 달리 '식재료'에 특화되어 있으며, 생산지가 아닌 '유통지(동네 마트)' 중심의 높은 접근성을 제공합니다666666666.

## 3. 핵심 기능 요구사항

### 3.1 상품 등록 및 관리

- **임박/B급 상품 등록:** 유통기한 임박 상품이나 외관상 흠이 있는 B급 식재료를 할인가로 등록합니다7.
- **자동 상태 관리:** 마감 시간(`deadline`) 경과 시 자동으로 노출이 중단되는 스케줄링 기능이 필요합니다888.

### 3.2 위치 기반 시스템

- **실시간 알림:** 상품 등록 시 설정된 반경 N(km) 내 사용자들에게 즉시 푸시 알림을 발송합니다9.
- **효율적 조회:** 사용자 근처의 마트 상품을 빠르게 찾을 수 있는 위치 기반 조회 최적화가 필요합니다10.

### 3.3 예약 및 주문

- **선착순 예약:** 알림을 받은 사용자가 앱을 통해 선착순으로 예약합니다11.
- **방문 수령:** 예약 성공 시 사용자가 직접 매장을 방문하여 포장 수령합니다12.

## 4. 기술 및 비즈니스적 도전 과제

### 4.1 기술적 과제

- **재고 동시성 제어:** 한정된 재고(예: 1개)에 대해 다수의 사용자가 동시에 예약 시도 시 데이터 무결성을 보장해야 합니다13131313.
- **상태 관리:** 마감 시간 엄수 및 실시간 재고 반영이 중요합니다14.

### 4.2 비즈니스적 과제

- **사장님 UX 최적화:** 오프라인 문화에 익숙한 어르신 사장님들을 위해 회원가입부터 등록까지의 Flow를 극단적으로 간소화해야 합니다15.
- **대안 타겟 확보:** 젊은 창업자가 많은 과일/반찬 가게 등으로 타겟을 확장하여 효용성을 증명합니다16.

## 5. 데이터 설계 및 API 요약

5.1 핵심 엔티티 17

- **USERS:** 사장님(OWNER)과 일반 사용자(USER)로 구분18.
- **STORE:** 가게 이름, 주소, 위치 정보(위도/경도) 포함 19.
- **PRODUCT:** 가격, 재고, 마감 시간, 상태(AVAILABLE, SOLD_OUT, EXPIRED) 관리 20.
- **ORDERS:** 예약 상태(RESERVED, PICKED_UP, CANCELED) 관리21.

  5.2 주요 API 엔드포인트 22

- `POST /api/v1/auth/sign-up`: 회원가입 23
- `POST /api/v1/stores`: 가게 등록 24
- `POST /api/v1/products`: 타임세일 상품 등록 25
- `GET /api/v1/products`: 내 주변 상품 조회 (위도, 경도, 반경 기반) 26
- `POST /api/v1/orders`: 상품 예약 (동시성 제어 필수) 27

- **사장님 전용 앱 편의성:** AI를 활용한 상품 등록 사진 인식 기능 등을 통해 등록 허들을 낮추는 것을 고려 안해잉~
